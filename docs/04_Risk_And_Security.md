# 潜在风险与防范 (Risk and Security)

生命体软件与传统自动化软件的最大差异在于其自我衍生能力，由此产生的最大风险在于“失控”。

## 1. 权限与审查
* **人工确认钩子 (Human-in-the-loop)**：在演进环修改核心逻辑或进行高危操作（如 Shell 环境调整）时，引入审批流程。必须发送如飞书/钉钉的异步通知，获得显式确认后才能落地。
* **分级授权机制**：对常规优化使用自动通过，对涉及到资源、系统配置变更实施阻断并申请审批。

## 2. 安全与隔离
* **沙箱化运行 (Sandboxing)**：所有的临时尝试、自动改写代码或插件更新，都必须且只能在限定资源与网络权限的沙箱中执行。
* **异物变异检测**：能够识别不符合安全范式的代码注入，免疫或隔离受损模块，避免系统级扩散。

## 3. 回滚与恢复
* **常态化快照**：“进化”或修改动作必须伴随代码与数据的快照，实现一键撤销（Rollback）到“出厂设置”或上一稳定版本的功能。
* **状态熔断**：如果检测到自演进导致的可用性降低或资源飙升（如大量创建无效垃圾容器），自动触发熔断策略，冻结系统变动能力。
