# HuiFlow 系统“生命体”演进：Node.js 全栈可落地架构蓝图

这是一份针对 `HuiFlow` 现存业务系统（非千万级并发的 B 端/项目协同平台），完全基于 **Vue 3 + Node.js (TypeScript)** 全栈生态打造“生命体软件”的实施总纲领。

核心宗旨：**前端组件化指令呈现，后端集约化 AI 调度，物理隔离底层引擎。以最小的开发阻力，实现系统对业务意图的动态生成与自愈特征。**

---

## 一、 系统架构整体视图 (The Macro Architecture)

我们将系统划分为物理隔离的三个层次：

### 1. 神经末梢层 (Sensory & Generative UI) - Vue 3 前端
不再执着于全栈 AI 生成 HTML/CSS。前端退化为高级的“组件积木拼装厂”与“感知雷达”。
*   **交互入口 (Copilot Panel)**：全局悬浮的对话面板，用户发出自然语言指令。
*   **响应器 (JSON Schema Renderer)**：引入 `@ai-sdk/vue` 接收后端流式数据。
    *   **展示类意图**：直接拿着预编译的 `<AutoChart>` 或 `<SmartTable>` 结合后端 Schema，在对话流中按需渲染，阅后即焚（Just-in-Time UI）。
    *   **操作类/复杂流转意图**：丢弃生成，响应后端下发的指令（如 `action: OPEN_TASK_MODAL`），通过 EventBus 自动唤起原有的复杂填表页面，并静默填充提取的上下文数据（Hybrid Mode）。
*   **无痕感知引擎**：埋点用户的无效高频点击和系统错误，静默打包上下文并上传至后端的“反思神经网”。

### 2. 调度与反思大脑 (The Agentic Brain) - Node.js (TypeScript) 中台
这是整个系统新建立的核心引擎（BFF - Backend For Frontend）。
*   **技术栈**：NestJS / Express (TypeScript) + Vercel AI SDK Core + LangChain.js。
*   **职责范围**：
    *   **语义解析路由 (Router Agent)**：判断用户的自然语言指令是“问数”、“复杂填报”还是“找茬（报错解答）”。
    *   **动态沙箱引擎 (Sandbox Executor)**：利用 Node.js 原生的 `vm` 模块，为大模型划定一个绝对安全隔离的内容空间，专门用来“当场执行”生成的胶水代码（如提取特殊格式的飞书日志数据）。即使代码死循环，也只限于当前沙箱上下文。
    *   **规则变异与自愈 (Rescuer Node)**：监听底层基座的异常日志，或者网关的限流报警，动态修改存储于 Redis 或 DB 的配置参数（限流阈值/路由备胎通道），并在下一毫秒生效，无需发版。

### 3. 持久化与业务基座 (The Static Muscle Engine) - Java 或 Go (未来平滑演进)
*   所有涉及核心账务、高并发发券、强一致事务、重型分布式调用的，**严禁引入任何大模型推断耗时。它是全静态编译的死逻辑。**
*   它向 Node.js 大脑层只暴露纯粹的 RESTful API/gRPC 接口。
*   **演进优势**：只要接口契约不变，未来如果你要抗下千万并发，随时可以把底层的某个模块重构成 Java 并发核心，而上层的 AI 大脑（Node.js 层）对底层重构完全无缝无感。这就叫“绞杀者微内核模式”。

## 二、 MVP 阶段行动计划 (30天内跑通闭环)

我们要用最短时间验证这个生态在 HuiFlow 中的价值。

### 里程碑 1：前端局部“呼吸感” (Generative UI 跑通)
*   **Action**：在现有的 Vue 代码里引入 Vercel AI SDK。
*   **Goal**：不碰任何老的业务页面。只做一张新页面/或者是全局侧边栏，让用户输入“昨天谁加班最多”，后台 Node.js 拿着数据拼好 Schema 丢回来，前端瞬间长出一张 `<柱状图>`。

### 里程碑 2：“操作员模式”混动跑通 (Hybrid Command Mode)
*   **Action**：定义极简指令协议（如 `OPEN_XXX_MODAL`）。
*   **Goal**：用户输入“帮我提个请假单”。Node 识别意图，返回指令。前端不生成任何页面，而是乖乖地把你历史手写的、极其稳定的“请假弹窗”拉起来。

### 里程碑 3：Node.js 运行时沙箱验证 (Dynamic Execution)
*   **Action**：找一个极其非标的需求（比如产品经理说：“我要把这篇飞书文档里的不规则文本抠出来生成待办，你们以前没做过这功能”）。
*   **Goal**：在 Node 环境里，Agent 当场写一段解析正则表达式的 JS 脚本，塞进 `vm` 沙箱里跑一遍获取出待办标题。不发版、不重启，解决随机的一批非标需求。

## 三、 对架构安全底线的 3 条铁律

如果你们真的上这套架构，必须和团队确立这三条规矩：

1.  **铁律一：大模型绝不上主干道**：核心的并发高可用路径（如写入数据库）只能由静态的 API 函数负责，大模型只能“组织并组装调用参数”，它绝不替原生业务层去手写 SQL 查询或直接插库。
2.  **铁律二：动态执行必进沙箱**：无论用哪家的模型生成出来的脚本，必须且只能塞进 Node.js 隔离的 `vm` 环境中执行，不能挂载文件系统(`fs`)，不能赋予特权网络代理(如绕开 SSRF 防火墙)。
3.  **铁律三：重交互必混动**：凡是表单级联超过 3 层、带有复杂联动校验的弹窗（占比 20%），必须由前端人肉手写固定组件，禁止 JSON Schema “硬刚生成”，极大概率渲染错误率极高。

---

**最终价值**：通过建立独立的 **TypeScript 全栈 AI 脑力中台**，HuiFlow 既保住了十年来沉淀的业务逻辑和复杂弹窗，又以极低门槛切入了“生成式响应、动态规则热插拔”的下一代软件形态。
